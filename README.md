# Tugas PBO Pertemuan Keduabelas
Koneksi IReport, Jasper, File CSV dan JFrame Form Netbeans dengan JPA (Java Persistence API) dan Dihubungkan dengan Foreign Key di Kontainer Netbeans.
#  JAVA PERSISTENCE API (JPA)
Java Persistence API (JPA) adalah spesifikasi resmi Java yang digunakan untuk memetakan data dari database relasional ke dalam bentuk objek Java (Object Relational Mapping/ORM). Dengan JPA, pengembang dapat mengelola, menyimpan, dan mengambil data dari database tanpa perlu menulis banyak kode SQL secara langsung.
JPA bekerja dengan cara menghubungkan kelas Java yang disebut Entity dengan tabel di dalam database. Setiap objek dari kelas tersebut mewakili satu baris data di tabel. Proses pengelolaan data dilakukan melalui EntityManager, yang bertugas untuk menambah, memperbarui, mencari, dan menghapus data.
Selain itu, JPA juga menggunakan JPQL (Java Persistence Query Language), yaitu bahasa query yang mirip dengan SQL tetapi bekerja menggunakan nama kelas dan atribut, bukan nama tabel dan kolom. Konfigurasi koneksi dan pengaturan entitas JPA biasanya disimpan dalam file persistence.xml yang mendefinisikan unit persistensi.
JPA hanya merupakan spesifikasi, sehingga untuk menggunakannya dibutuhkan implementasi seperti Hibernate, EclipseLink, atau OpenJPA. Dengan JPA, pengembangan aplikasi menjadi lebih efisien karena data dapat diolah langsung melalui objek Java tanpa perlu berinteraksi langsung dengan perintah SQL, serta mendukung berbagai sistem database dengan sedikit perubahan kode.
# PRIMARY KEY DAN FOREIGN KEY
  Primary Key adalah atribut atau kombinasi atribut pada sebuah tabel yang berfungsi untuk membedakan setiap baris data agar bersifat unik dan tidak boleh kosong. Primary key digunakan sebagai identitas utama suatu record, sehingga tidak ada dua data yang memiliki nilai primary key yang sama.
Foreign Key adalah atribut pada suatu tabel yang digunakan untuk menghubungkan data dengan tabel lain melalui referensi ke primary key tabel tersebut. Foreign key berfungsi untuk menjaga hubungan antar-tabel dan memastikan konsistensi serta integritas data di dalam basis data.
# KONTAINER GUI 
Kontainer dua tab pada NetBeans GUI menggunakan komponen JTabbedPane yang berfungsi untuk menampilkan beberapa panel dalam satu jendela aplikasi. Melalui komponen ini, pengguna dapat berpindah antar-tab dengan mudah tanpa membuka form baru. Setiap tab berisi tampilan atau fungsi yang berbeda, misalnya tab pertama menampilkan data pemilik, sedangkan tab kedua menampilkan data hewan. Dengan penggunaan JTabbedPane, tampilan aplikasi menjadi lebih rapi, terstruktur, dan efisien karena semua fitur dapat diakses dalam satu frame utama. Selain itu, komponen ini juga memudahkan pengelompokan data atau fitur yang saling berhubungan, sehingga meningkatkan kenyamanan dan kemudahan bagi pengguna dalam mengoperasikan aplikasi.
# Langkah-langkah
1. Persiapan Awal. Buka NetBeans IDE dan buat Project Java baru. Pastikan sudah terpasang PostgreSQL dan pgAdmin di komputer. Buat database baru di pgAdmin dengan klik kanan pada Databases → Create → Database, beri nama sesuai kebutuhan, lalu klik Save.
2. Koneksi Database ke NetBeans. Di NetBeans, buka tab Services → Databases → Drivers. Klik kanan pada PostgreSQL → Connect Using. Isi nama database dan password PostgreSQL kamu. Klik Test Connection untuk memastikan koneksi berhasil. Salin JDBC URL yang akan digunakan untuk konfigurasi.
3. Membuat Entity Classes from Database (JPA). Klik kanan pada package proyek → New → Entity Classes from Database. Pilih Data Source sesuai database PostgreSQL yang sudah dibuat. Pindahkan tabel yang diinginkan ke kolom kanan (Available Tables → Selected Tables). Centang Generate Named Query Annotation for Persistent Field dan Use Column Names in Relationship. Klik Finish untuk membuat entity classes dan file persistence.xml secara otomatis.
4. Menambahkan Library. Klik kanan pada Libraries di project. Tambahkan: PostgreSQL JDBC Driver (Add Library → pilih PostgreSQL Driver). JasperReports Library (Add JAR/Folder → pilih file Jasper .jar).
5. Membuat Report dengan JasperReport. Klik kanan pada package → New → Report Wizard. Pilih ukuran kertas (A4, Letter, dll). Beri nama file .jrxml dan klik Next. Pilih NetBeans Database JDBC Connection, lalu pilih koneksi database kamu. Klik Test Connection untuk memastikan koneksi berhasil. Pindahkan semua fields ke kanan → klik Finish. File .jrxml dan .jasper akan otomatis dibuat.
6. Membuat GUI (JFrame Form). Klik kanan pada package → New → JFrame Form. Buat desain form menggunakan JTabbedPane untuk memisahkan tab “Pemilik” dan “Hewan”. Tambahkan komponen tombol: Insert, Update, Delete, Upload CSV, dan Cetak Jasper.
7. Membuat JDialog untuk CRUD. Buat 3 JDialog Form: InsertDialog → untuk menambah data. UpdateDialog → untuk mengedit data. DeleteDialog → untuk menghapus data. Tambahkan method: koneksi() → untuk menghubungkan ke database. bersih() → untuk membersihkan input form. simpan(), update(), hapus() → sesuai fungsinya.
8. Menghubungkan Button CRUD. Pada JFrame utama, buat aksi tombol: Insert → membuka InsertDialog. Update → membuka UpdateDialog. Delete → membuka DeleteDialog. Hubungkan setiap dialog dengan Entity JPA agar data tersimpan langsung di database.
9. Menambahkan Fungsi Upload File CSV. Tambahkan button Upload di JFrame. Gunakan FileChooser untuk memilih file CSV. Parse isi CSV dan masukkan datanya ke tabel database melalui JPA. Tampilkan notifikasi “Upload Selesai” jika berhasil.
10. Menambahkan Fungsi Cetak Laporan (JasperReport). Tambahkan button Cetak di JFrame. Hubungkan dengan file .jasper yang sudah dibuat. Gunakan JasperFillManager.fillReport() untuk mengisi data dari database. Tampilkan hasilnya menggunakan JasperViewer.
11. Membuat Main Class. Buat file Main.java. Atur Look and Feel menggunakan Nimbus. Jalankan JFrame utama (new JualBeliHewan().setVisible(true);). Aplikasi siap dijalankan.
